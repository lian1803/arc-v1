## ✅ 검증 1 — 광고 집행 전 3대 진입 조건 체크리스트

### [현재 조건에서 사용 가능한가: **조건부 YES**]

**리스크:**
- "아이템위너 점유 가능성 60% 이상"이라는 기준값이 **어떤 데이터를 근거로 산출되는지 기준이 없음**
- 신규 셀러의 경우 점유율 사전 예측 자체가 어려움 → 기준값이 공허한 숫자로 전락할 위험
- "가격·배송조건 조정 선행"이라는 미충족 시 조치가 **셀러에게 요구 사항을 돌리는 구조** → 계약 이탈 포인트

**현재 조건에서 보완할 것:**
- 점유 가능성 60% 기준을 **쿠팡 카테고리 공개 데이터(베스트100 기준가·배송조건) 기반으로 판단 가능한 체크리스트 형태로 전환**
- 예시: "현재 최저가 대비 ±5% 이내 가격 설정 가능한가 Y/N", "로켓배송 또는 익일 배송 가능한가 Y/N" → 3문항 이내로 압축해 셀러가 직접 자가진단 가능하게 설계
- 미충족 시 조치를 "팀이 함께 조건 검토"로 표현 전환 → 셀러가 혼자 해결해야 한다는 인상 제거

## ✅ 검증 2 — ROI 수치 및 성과 데이터 선제 공개 전략

### [현재 조건에서 사용 가능한가: **NO**]

**리스크:**
- 자사 실적 데이터가 **단 한 건도 없는 상태**에서 ROAS, 전환율, 매출 증가율 등 수치를 제시하면 **법적 허위 광고 리스크 + 셀러 신뢰 붕괴** 동시 발생
- 실제 계약 셀러가 없으므로 "평균 ROAS 3.2배" 류의 수치는 **근거 없는 보장**으로 간주될 수 있음
- 고객 이탈 포인트: 셀러가 "이 수치 어디서 나온 거냐"고 물었을 때 답변 불가 → 신뢰 0으로 수렴

**현재 조건에서 보완할 것:**
- 자사 수치 대신 **쿠팡 공식 발표 데이터 + 업계 리포트 수치를 출처 명시 후 인용**
  - 예: "쿠팡 2024 IR 자료 기준, 로켓배송 상품 전환율 일반 배송 대비 2.1배" (출처 명기)
- 수치 제시 방식 전환: "우리가 만든 성과" → "이 구조로 운영하면 시장 평균 대비 이 구간의 성과가 가능하다"는 **구조 설명형 제시**
- 첫 계약 셀러에게는 "파일럿 운영 → 실제 데이터 공동 수집" 프레임 적용 → 수치 없음을 약점이 아닌 **공동 검증 기회**로 전환

## ✅ 검증 3 — 신뢰 설계 전략 (레퍼런스 부재 상황)

### [현재 조건에서 사용 가능한가: **조건부 YES**]

**리스크:**
- 레퍼런스가 없을 때 가장 흔한 실수: "곧 성과를 보여드리겠다"는 **미래형 약속 남발** → 오히려 불안감 증폭
- 신규 서비스라는 사실을 숨기려는 시도 자체가 발각 시 신뢰 붕괴의 배수 효과
- 팀원 경력이 없을 경우 **"팀"의 신뢰가 아닌 "개인" 신뢰도 설계조차 불가**

**현재 조건에서 보완할 것:**
- 레퍼런스 부재를 **역이용하는 신뢰 설계** 3단 구조 적용:
  1. **투명성 선언**: "저희는 신설팀입니다. 그래서 첫 파트너 셀러에게 모든 운영 과정을 실시간 공유합니다"
  2. **과정 공개**: 주간 리포트 제공 약속 → 결과가 아닌 과정을 신뢰 자산으로 전환
  3. **리스크 분산 계약**: 단기 파일럿(4주) 후 연장 여부 셀러 결정 구조 → 셀러의 계약 리스크 체감 낮춤
- 팀원 개인 역량(쿠팡 카테고리 분석 능력, 광고 세팅 전문성 등)을 **포트폴리오 자료 없이 시연 가능한 형태로 준비** → 미팅 자리에서 실시간 분석 시연으로 대체

## ✅ 검증 4 — 차별화 논리 (경쟁사 대비)

### [현재 조건에서 사용 가능한가: **NO → 재설계 필요**]

**리스크:**
- 현재 전략의 차별화 포인트가 **"광고 최적화 잘 해드립니다"** 수준에서 벗어나지 못함
- 쿠팡 광고 대행은 이미 수십 개 업체가 운영 중 → 동일한 언어로 경쟁하면 가격 경쟁으로 수렴
- 신설팀이 가격 경쟁에서 이길 수 없음 → 차별화 논리 없으면 수주 불가

**현재 조건에서 보완할 것:**
- 차별화 축을 **"광고 잘 하는 팀"에서 "쿠팡 구조를 셀러보다 먼저 읽는 팀"으로 전환**
  - 구체화: 매주 카테고리별 아이템위너 점유 변동 모니터링 리포트를 셀러에게 제공 → 경쟁사 움직임을 셀러보다 먼저 알려주는 구조
- 신설팀만 가능한 차별화 포인트 활용: **"대형 대행사는 셀러 한 명에게 시간을 못 씁니다. 저희는 지금 한 셀러에 집중합니다"** → 소수 정예·집중 케어를 차별화 언어로 전환
- 첫 3개 셀러에게는 **경쟁사 광고 분석 리포트 무료 제공** (쿠팡 검색 노출 구조 분석 + 위너 점유 현황) → 계약 전 가치 선제 제공으로 신뢰 선점

## ✅ 검증 5 — 계약 리스크 설계 (셀러 관점)

### [현재 조건에서 사용 가능한가: **NO**]

**리스크:**
- 현재 제안서에 **계약 조건·해지 조항·성과 미달 시 처리 방식**이 명시되어 있지 않음
- 레퍼런스 없는 신규 팀과의 계약에서 셀러가 가장 우려하는 것: "성과 없으면 어떻게 되나" → 이 질문에 답이 없으면 계약 보류
- 성과 보장형 계약 남발 시 팀 내 재정 리스크 역발생

**현재 조건에서 보완할 것:**
- 계약 구조를 **3단계 안전망 설계**로 명문화:
  1. **4주 파일럿**: 월 고정비 최소화 (운영비
