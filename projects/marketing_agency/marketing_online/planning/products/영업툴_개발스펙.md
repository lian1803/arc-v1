## 0. 현재 코드 현황 (먼저 파악)

### 존재하는 것 (재사용 가능)

| 코드 | 위치 | 역할 | 재사용 방식 |
|------|------|------|-----------|
| `naver_crawler.py` | `company/utils/` | 스마트스토어/쿠팡 셀러 크롤링 (Playwright) | 잠재 클라이언트 브랜드 발굴 |
| `meta_ads.py` | `company/utils/` | Meta Ad Library API 경쟁사 광고 분석 | 클라이언트 광고 현황 진단 |

| 코드 | 이유 |

| `sales_crm` (오프라인팀) | 오프라인 소상공인 CRM. UI/데이터 구조 전혀 다름 |

## 1. 전체 영업 워크플로우

```
[잠재 클라이언트 발굴]
서진혁 크롤러 (naver_crawler.py) → 이커머스 브랜드 리드 카드 생성
                                  ↓
[브랜드 진단] (신규 개발: online_brand_diagnosis.py)
meta_ads.py → 광고 집행 현황 분석
naver_crawler.py → 스마트스토어/쿠팡 운영 현황 수집
→ 진단 리포트: 점수 + 문제 TOP3 + 패키지 추천
                                  ↓
[제안서 생성] (신규: 에이전시용 최제안 에이전트)
70만/330만/800만 패키지 기준
html_templates → PDF 자동 생성
                                  ↓
[계약/온보딩] (신규: CRM JSON 기반)
클라이언트 등록 → 팀 인계 → 월간 보고 스케줄
```

## 2. 신규 개발: online_brand_diagnosis.py

### 목적
오프라인 naver-diagnosis 대신, 이커머스/D2C 브랜드 진단 전용 모듈.

### 진단 항목 (기존 코드 활용)

```python
# naver_crawler.py 재사용
crawler = NaverCrawler()
store_data = crawler.analyze_smartstore(smartstore_url)
# → 팔로워, 상품 수, 리뷰 수, 평점, 상세페이지 품질

# meta_ads.py 재사용
from utils.meta_ads import spy
ads_data = spy(brand_name)
# → 광고 집행 여부, 광고 소재, 집행 기간, 경쟁사 대비 볼륨

# 추가 수집 (신규 구현)
- 인스타그램 팔로워 수 / 최근 포스팅일 (insta_browse.py 재사용)
- 구글 검색 상위 노출 여부 (requests 기반 간단 체크)
- 쿠팡 입점 여부 + 리뷰 수 (playwright)
```

### 점수 계산 (100점 만점)

| 항목 | 배점 | 기준 |
|------|------|------|
| 퍼포먼스 광고 | 30점 | 메타/구글 집행 여부, 예산 규모 |
| SNS 운영 | 25점 | 인스타 팔로워, 최근 포스팅 주기 |
| 이커머스 최적화 | 25점 | 스마트스토어 리뷰 수, 상세페이지 품질 |
| SEO/콘텐츠 | 20점 | 구글 노출, 블로그 운영 여부 |

### 출력 형식

```
[브랜드명] 온라인 마케팅 진단
━━━━━━━━━━━━━━━━━━━━━━━
종합 점수: 42/100 (D등급)

[지금 당장 손해 보고 있는 3가지]
1. 메타 광고 미집행 → 경쟁 브랜드 대비 인지도 격차 월 단위로 벌어짐
2. 인스타 최근 포스팅 45일 전 → 팔로워 이탈 가속
3. 스마트스토어 리뷰 7개 → 전환율 업계 평균 대비 추정 -40%

[추천 패키지]
→ Growth Plus (330만원/월) — 광고+SNS+콘텐츠 동시 해결
━━━━━━━━━━━━━━━━━━━━━━━
```
