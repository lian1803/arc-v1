# 영업툴 개발 스펙서
> 작성일: 2026-04-18 | 코드팀(core-shell) 전달용

---

## 개요

영업사원이 소상공인 1곳을 10분 안에 진단 → 제안서 생성 → 계약까지 이어지는
원스톱 워크플로우를 자동화하는 툴 3종.

```
[영업사원 워크플로우]
업체 발견 → naver-diagnosis 진단 → 제안서 PDF 자동생성 → DM 발송 → CRM 등록 → 계약 → 온보딩
```

---

## 툴 1: naver-diagnosis (현재 미완성 → 완성 필요)

### 현재 상태
- FastAPI 서버 구동 가능
- 기본 크롤링 + 점수 계산 있음
- **미연결 모듈**: photo_quality_score, review_sentiment_score, ai_analyzer.py, photo_analyzer.py
- **미수집**: 리뷰 응답률, 소식 최신성, 경쟁사 비교

### 완성 요구사항 (Priority 순)

#### P0 — 반드시 있어야 영업 가능
- [ ] 업체명 입력 → 네이버 플레이스 자동 크롤링 (현재 있음, 안정화 필요)
- [ ] 진단 점수 100점 계산 (정보 완성도 50 + 리뷰 30 + 사진 20)
- [ ] **경쟁사 2~3곳 자동 비교** (같은 동네, 같은 업종 상위 3곳)
- [ ] 진단 리포트 PDF 자동 생성 (현재 HTML 있음 → PDF export 필요)
- [ ] 리포트 내용: 현재 점수 / 경쟁사 점수 비교 / 개선 항목 3가지

#### P1 — 있으면 훨씬 강해짐
- [ ] Gemini Vision 사진 품질 분석 연결 (photo_analyzer.py 이미 있음)
- [ ] 리뷰 감성 분석 (review_sentiment_score 모델에 있음, scorer 미연결)
- [ ] 개선 후 예상 순위 변화 제시 ("이 업종 지역에서 현재 8위 → 개선 시 3위 예상")
- [ ] 리포트 커버 페이지에 업체명/날짜 자동 삽입 (PDF 퀄리티)

#### P2 — 나중에
- [ ] 배치 진단 (엑셀 업체명 리스트 → 일괄 진단)
- [ ] 진단 이력 DB 저장 + 재진단 비교
- [ ] 영업사원 로그인 → 자기 담당 업체만 보기

### 출력물 스펙 (PDF 1장)
```
┌─────────────────────────────┐
│  [업체명] 네이버 플레이스 진단  │
│  2026-04-18                 │
├─────────────────────────────┤
│  종합 점수: 38 / 100         │
│  업종 평균: 61 / 100         │
│  상위 경쟁사: [경쟁사명] 84점  │
├─────────────────────────────┤
│  개선 필요 항목              │
│  1. 사진 (현재 3장 → 20장 필요)│
│  2. 소식 (0개 → 월 4회 필요)  │
│  3. 리뷰 응대율 (12% → 80%)  │
├─────────────────────────────┤
│  개선 후 예상 점수: 72 / 100  │
│                             │
│  문의: 리안 컴퍼니            │
│  [담당자 이름 / 연락처]        │
└─────────────────────────────┘
```

## 툴 2: sales_crm (현재 있음 → 패키지 연동 필요)

### 추가 요구사항

#### P0
- [ ] 업체 등록 시 **패키지 선택 필드** (A/B/C — 38만/56만/95만)
- [ ] 계약 상태 필드: 진단중 / 제안전달 / 상담중 / 계약완료 / 운영중 / 해지
- [ ] 월 청구 금액 자동 계산 (패키지 기준: A=380,000 / B=560,000 / C=950,000)
- [ ] 담당자 배정 필드

#### P1
- [ ] naver-diagnosis 진단 이력 연결 (업체명 기준 자동 매칭)
- [ ] 다음 액션 알림 (D+1 권한 요청 / D+7 첫 보고 / D+30 월 리포트)
- [ ] 월 리포트 자동 생성 버튼 (진단 점수 변화 + 리뷰 수 변화)
- [ ] 업셀 제안 타이밍 알림 (A → 2개월차, B → 3개월차)

#### P2
- [ ] 계약서 PDF 자동 생성 (업체명, 패키지, 기간, 금액 자동 입력)
- [ ] 카카오 알림 연동 (다음 액션 일자 알림)

## 기술 현황 (현재 naver-diagnosis)

```
naver-diagnosis/
├── main.py          ← FastAPI 진입점
├── app.py           ← (없음 — main.py가 app 역할)
├── database.py      ← SQLite DB
├── models.py        ← photo_quality_score, review_sentiment_score 있음 (미연결)
├── services/        ← 비즈니스 로직
├── routers/         ← API 라우터
├── html_templates/  ← 리포트 HTML 있음
├── browser_manager.py ← Playwright 관리
├── services/
│   ├── ai_analyzer.py    ← Gemini 분석 (구현됨, 미호출)
│   └── photo_analyzer.py ← 사진 분석 (구현됨, 미호출)
└── generate_html_pdf.py  ← PDF 생성 (있음, 완성도 미확인)
```
