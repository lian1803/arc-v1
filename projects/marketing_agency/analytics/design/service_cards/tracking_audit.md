# 트래킹 감사 (Tracking Audit)

---

## 이게 왜 필요한가

광고를 오래 돌리다 보면 트래킹이 조용히 망가진다. 누군가 사이트를 수정하면서 픽셀이 중복됐거나, 이벤트가 깨졌거나, 구매 완료 페이지에서만 이벤트가 2번 발화되거나. 겉으로는 데이터가 쌓이는 것 같아서 모른다.

트래킹 감사는 지금 우리 데이터가 얼마나 믿을 수 있는지를 점검하는 작업이다.

트래킹 감사가 필요한 신호:
- 광고 플랫폼 전환 수와 실제 주문 수가 계속 안 맞음
- 메타 광고 ROAS가 갑자기 올라갔는데 실제 매출이 그만큼 안 나옴
- GA4 페이지뷰가 갑자기 2배 뛰었음
- 사이트 리뉴얼이나 이벤트 추가 작업 후 이상한 숫자가 보임
- "우리 데이터 믿어도 되나?" 라는 의심이 드는데 확인할 방법을 모름

---

## 우리가 해주는 것

감사 항목:

**GA4**
- 측정 ID 중복 여부 확인 (이중 집계 체크)
- 주요 이벤트 발화 정상 여부 (구매, 장바구니, 폼 제출 등)
- 이벤트 파라미터 정확도 (금액, 상품명, 수량 정보 포함 여부)
- 데이터 보존 기간 설정 확인
- 세션 / 전환 데이터 비정상 패턴 탐지

**메타 픽셀 + CAPI**
- 픽셀 설치 중복 여부
- purchase 이벤트 정상 발화 확인
- CAPI 설치 여부 및 이중 추적 중복 제거 상태
- 이벤트 매치 품질(EMQ) 점수 확인

**GTM**
- 태그 충돌 여부 (같은 이벤트가 여러 태그에서 발화되는지)
- 트리거 설정 오류 탐지
- 변수 누락 또는 잘못된 값 전달 여부

**결과물**
- 감사 리포트 (문제 항목 + 심각도 등급 + 수정 권고사항)
- 스탠다드 이상: 수정 작업 포함

---

## 없으면 어떤 일이 벌어지나

- 오염된 데이터로 광고 최적화를 계속 함 → 알고리즘이 엉뚱한 방향으로 학습
- ROAS 10이라고 믿고 예산을 2배로 늘렸는데 실제로는 5였던 경우 → 예산 낭비
- 이미 몇 달간 이중 집계된 데이터로 트렌드 분석을 해온 경우 → 모든 분석 신뢰 불가
- 사이트 개편 후 픽셀이 끊겼는데 모르고 3개월 동안 광고 최적화 신호 없이 운영

---

## 가격

| 패키지 | 감사 포함 여부 | 감사 주기 | 월 가격 |
|--------|--------------|----------|---------|
| 스타터 | 분기 1회 (리포트만) | 3개월 1회 | 50~80만원 |
| 스탠다드 | 분기 2회 (리포트 + 수정 권고) | 1.5개월 1회 | 120~180만원 |
| 프리미엄 | 월 1회 (리포트 + 수정 작업 포함) | 매월 | 250~350만원 |

단독 1회성 트래킹 감사: 별도 견적 (규모에 따라 상이, 통상 30~80만원).

---

## 소요 기간

- 기본 감사 (GA4 + 픽셀): 2~3 영업일
- 전체 감사 (GA4 + 픽셀 + CAPI + GTM): 5~7 영업일
- 감사 결과 기반 수정 작업: 문제 규모에 따라 추가 3~10 영업일

---

## 주의사항

- 감사는 리드오프 → 실측 확인 방식으로 진행 (Tag Assistant, Pixel Helper, GA4 DebugView 등 활용)
- 사이트 접근 권한 필요 없음 — 공개된 프론트엔드 기준 확인 후, 백엔드 수정 필요 시 개발자 협조 요청
- 감사 결과는 문제 리스트 + 수정 우선순위 형태로 제공 (직접 수정 포함 여부는 패키지에 따라 다름)
- 수정 작업은 사이트 개발자 또는 GTM 계정 접근 권한 필요
- 감사 완료 후 데이터 안정화까지 최소 72시간 확인 권장
