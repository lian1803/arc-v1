# 히디프(HIDIFF) 트래킹 인프라 구축 제안서

**수신:** 이휴림 대표님
**발신:** LIAN 마케팅 에이전시 데이터분석팀
**일시:** 2026-04-19
**제안 패키지:** 스탠다드

---

## 지금 이 순간에도 손실이 발생하고 있습니다

> 화해 뷰티어워드 3관왕. 인스타그램 팔로워 20,300명.
> 히디프는 지금 **성장할 준비가 된 브랜드**입니다.
> 그런데 그 성장이 **어디서 오는지 측정되지 않고 있습니다.**

히디프 사이트에는 GA4, GTM, 메타 픽셀이 모두 설치되어 있습니다.
그러나 저희 진단 결과, **지금 이 세 도구가 수집하는 데이터는 신뢰할 수 없는 상태**입니다.

- GA4 트래킹 ID가 **2개 동시 작동 중** → 세션·전환 수치가 **실제보다 부풀려져** 집계되고 있습니다
- 전환 이벤트는 **단 1개(`conversion`)만 측정 중** → 고객이 장바구니에 담고 이탈하는지, 결제 단계에서 막히는지, 어떤 상품이 팔리는지 **아무것도 보이지 않습니다**
- 메타 픽셀은 설치되어 있지만 **구매 금액·상품 정보가 전달되지 않아** 지금 집행 중인 광고의 실제 ROAS가 **0원으로 잡히고 있습니다**

광고비를 집행하고, 수상 효과로 유입이 늘었는데 —
**"어떤 채널이 매출을 만들었는가"에 대한 답이 지금 없습니다.**

## 현황 진단 요약

| 진단 항목 | 상태 | 현재 발생 중인 문제 |
|-----------|------|---------------------|
| GA4 설치 | ⚠️ 2개 중복 감지 (`G-07V7RKPJ1M` / `G-598429`) | 세션·전환 이중 집계 → 모든 지표 기준값 오염 |
| GTM 설치 | ✅ 감지됨 (`GTM-PJ6QB7NT`) | 설치는 됐으나 내부 태그 정리 미완료 |
| Meta Pixel | ⚠️ 설치됨, 이벤트 미연동 (`483166674335780`) | 광고 최적화 신호 없음 → ROAS 실측 불가 |
| dataLayer | ⚠️ 감지됨, 구조화 미완료 | 상품명·가격·주문번호 등 속성값 비어있음 |
| 전환 이벤트 | ⚠️ 1개만 존재 (`conversion`) | 퍼널 전 구간 데이터 공백 |
| 필수 이커머스 이벤트 | ❌ 미측정 | `view_item` / `add_to_cart` / `begin_checkout` / `purchase` 전부 미감지 |
| 스크립트 오류 | ✅ 없음 | — |

> **핵심 판정:** 도구는 설치됐지만 데이터는 수집되지 않고 있습니다.
> 지금 GA4 대시보드의 숫자는 **의사결정에 사용할 수 없는 상태**입니다.

## 이 손실의 구체적 비용

민재 채점 결과(성숙도 등급 2 — "설치형, 미검증")를 기반으로 현재 발생 중인 기회손실을 구조화했습니다.

### 손실 유형 1 — 광고비 최적화 신호 부재

메타 픽셀에 구매 이벤트(`Purchase`)와 구매 금액(`value`)이 전달되지 않으면, 메타 광고 알고리즘은 **"이 광고가 매출을 만들었는가"를 학습하지 못합니다.** 알고리즘은 현재 `conversion`이라는 불분명한 이벤트를 기준으로 최적화 중입니다. 이 상태에서는:

- 실제로 구매로 이어지지 않는 유저에게 광고가 반복 노출됩니다
- 잘 팔리는 상품 페이지에 예산이 집중되지 않습니다
- **광고비 중 최소 20~30%가 전환 가능성이 낮은 노출에 소진**되고 있는 구조입니다

### 손실 유형 2 — GA4 이중 집계로 인한 판단 오류

`G-598429`는 GA4 표준 ID 형식(G-XXXXXXXXXX 10자리)에 맞지 않는 비정상 ID입니다. 이 ID가 동시에 발화하면서 **세션·전환 수치가 실제보다 높게 집계**됩니다. 이로 인해:

- 전환율이 실제보다 낮게 계산됩니다 (분자인 전환은 1배, 분모인 세션은 2배)
- 또는 반대로 특정 전환이 2회로 기록되어 ROAS가 부풀려집니다
- **지금까지의 의사결정 근거가 된 수치 전체가 재검토 대상**입니다

### 손실 유형 3 — 퍼널 이탈 원인 불명

`add_to_cart`와 `begin_checkout`이 미측정 상태입니다. 고객이 상품 페이지에서 이탈하는지, 장바구니에서 이탈하는지, 결제 단계에서 이탈하는지 **구분이 불가합니다.** 화해 수상 이후 유입이 증가했다면, 이 유입이 실제 구매로 전환되는지 여부도 **지금은 검증할 방법이 없습니다.**

> 인스타 팔로워 2만 명, 화해 뷰티어워드 3관왕 브랜드가 지금 가장 먼저 해야 할 일은

스타터 패키지는 신규 설치 중심으로, 히디프처럼 **기존 설치를 전면 재구성**해야 하는 케이스에는 작업 범위가 부족합니다. 프리미엄 패키지는 현재 데이터 기반 자체가 오염된 상태에서 투입하면 인풋이 잘못된 고급 분석이 됩니다. **지금 히디프에 맞는 패키지는 스탠다드입니다.**

### 포함 작업 전체 목록

#### Phase 1 — 기반 정비 (1~2주차)

| 작업 | 내용 | 이것이 없으면 |
|------|------|---------------|
| **GA4 중복 ID 제거** | `G-598429` 구버전 ID 제거, `G-07V7RKPJ1M` 단일 운영 체계 확립, GTM 내 트리거 정리 | 모든 데이터의 기준값이 오염된 채로 유지됨 |
| **GA4 이커머스 이벤트 5종 세팅** | `view_item` / `add_to_cart` / `begin_checkout` / `add_payment_info` / `purchase` — GTM dataLayer 기반 구현 | 퍼널 어디서 이탈하는지 영원히 모름 |
| **dataLayer 이커머스 스키마 설계** | 상품 ID·상품명·카테고리·가격·수량·주문번호 포함 구조 정의 + 개발팀 전달용 스펙 문서 작성 | 이벤트를 심어도 "어떤 상품이 팔렸는지" 속성값이 비어 분석 불가 |
| **Meta Pixel 구매 이벤트 연동** | `Purchase` 이벤트에 `value` + `currency` 파라미터 추가, GA4 purchase와 동일 dataLayer 활용 | 메타 광고 ROAS가 실측 0원 상태 → 알고리즘 최적화 신호 없음 |
| **GA4 전환 이벤트 재설정** | 기존 `conversion` 이벤트 검토 후 `purchase`를 핵심 전환으로 등록, `begin_checkout` 보조 전환 추가 | 광고 최적화 알고리즘이 엉뚱한 행동을 전환으로 학습 |
| **GTM 컨테이너 정리** | 미사용 태그·트리거·변수 감사(Audit) 후 제거, 네이밍 규칙 통일 | 태그 충돌·중복 실행으로 이벤트 오발화 지속 |

#### Phase 2 — 멀티채널 확장 (3~4주차)

| 작업 | 내용 |
|------|------|
| **Meta CAPI(서버사이드) 구축** | iOS 14+ 이후 브라우저 기반 픽셀로 잡히지 않는 전환을 서버에서 직접 수집 — 전환 추적 커버리지 회복 |
| **구글 애즈 전환 태그 고급 설정** | `purchase` 이벤트 기반 전환값 전달, 향상된 전환(Enhanced Conversions) 세팅 |
| **네이버 광고 전환 추적 세팅** | 네이버 쇼핑·검색광고 전환 태그 GTM 기반 설치 |
| **카카오 픽셀 / 카카오 모먼트 세팅** | 카카오 채널 운영 시 전환 데이터 수집 체계 구축 |

#### Phase 3 — 리포트·대시보드 (5주차~운영)

| 구성 | 내용 |
|------|------|
| **대표님용 대시보드** | 월별 매출·채널별 ROAS·광고비 대비 실적 한눈에 |
| **마케터용 대시보드** | 일별 채널 지출·클릭·CPC·전환율 실시간 모니터링 |
| **퍼널 분석 대시보드** | view_item → add_to_cart → begin_checkout → purchase 단계별 이탈률 추적 |
| **월별 성과 리포트** | 채널별 성과 분석 + 다음 달 예산 배분 제안 포함 (매월 5일 발행) |
| **분기별 데이터 품질 감사** | 태그 오발화·이벤트 누락·중복 집계 여부 정기 점검 (연 2회) |
| **월 1회 데이터 분석 컨설팅** | 30분, 화상 또는 대면 — 리포트 기반 액션 플랜 도출 |
