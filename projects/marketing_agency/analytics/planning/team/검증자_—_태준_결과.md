### [V-01] 설계 근거 테이블 정합성 검증

**검증 대상:** 전제 조건 요약 테이블 (발견 결함 4개 항목)

```
검증 항목:
- "GA4 ID 2개 중복 설치"
- "전환 이벤트 1개만 존재 (conversion)"
- "이커머스 dataLayer 미구조화"
- "view_item / add_to_cart / purchase 미측정"
```

**문제 탐지:**

이 4개 항목은 유진·민재 진단 결과를 반영했다고 명시되어 있으나, **제출된 문서 내에 원본 진단 수치가 없습니다.** 진단 원본이 검증되지 않은 상태에서 설계서가 이를 "확정 사실"로 기술하고 있습니다.

- `"세션·전환 수치 2배 부풀려짐"` — 2배라는 수치의 출처 불명. GA4 중복 태그가 세션을 정확히 2배로 계산하는 것은 **조건부 사실**입니다. GTM 실행 조건, 태그 타입에 따라 실제 중복 계수는 1.3배~2배 사이 편차가 존재합니다. "2배"를 단정 표기하면 클라이언트 현황과 불일치 시 신뢰도 타격이 됩니다.

```
현재 조건에서 사용 가능한가: 조건부YES
리스크: "2배 부풀려짐" 단정 표기 → 실측값과 불일치 시 제안서 전체 신뢰도 훼손
현재 조건에서 보완할 것:
  → "2배" → "최대 2배 수준으로 과집계될 수 있음"으로 표현 완화
  → 진단 원본(유진·민재) 수치 인용 시 출처 명시 각주 추가
  → 보완 위치: 전제 조건 요약 테이블 2행 비즈니스영향 셀
```

### [V-02] 패키지 구성 — 범위 확인

**문제 탐지:**

설계서에 `PACKAGE A — 기반 정비 패키지` 제목이 존재하나, **패키지 A의 세부 구성 항목(deliverable 목록, 작업 범위, 납기, 금액)이 이 문서에 기재되지 않았습니다.** 문서가 중간에 절단된 것으로 판단됩니다.

- 패키지 B, C 존재 여부 불명
- 단가, 총액, 납기 일정 없음
- 하윤 작가 문서(제안서)와 패키지명 대조 필요: 하윤 문서는 "스탠다드"를 선택했다고 기재 → 설계서에서 "스탠다드"에 해당하는 패키지가 어느 것인지 매핑 불가

```
현재 조건에서 사용 가능한가: NO
리스크:
  - 패키지 본문 누락으로 제안서(하윤)와 연결 불가
  - "스탠다드 = PACKAGE A" 매핑 근거 없음 → 영업 현장에서 혼선 발생
현재 조건에서 보완할 것:
  → 패키지 A 세부 deliverable 최소 항목: ① GTM 태그 정리 범위 ② GA4 단일화 작업 ③ dataLayer 구조화 항목 ④ 납기 주 단위 명시 ⑤ 단가
  → "스탠다드" 명칭과 PACKAGE A/B/C 간 매핑 테이블 1행 추가
  → 보완 위치: PACKAGE A 섹션 본문 (현재 "대상:" 이하 내용 없음)
```

현재 조건에서 보완할 것:
  → 패키지 본문 보완 후 재검증 필요
  → 재검증 요청 항목: 성과 수치 약속 문구, SLA 조건, 환불 조건 유무

### [V-04] 제안서 ↔ 설계서 수치 정합성 대조

**검증 대상:** 제안서 본문 내 기술 사실 항목

| 제안서 기재 내용 | 설계서 대응 항목 | 정합 여부 |
|-----------------|----------------|----------|
| "GA4 트래킹 ID가 2개 동시 작동 중" | 설계서 V-01 동일 | ✅ 일치 |
| "세션·전환 수치가 실제보다 부풀려져 집계" | 설계서 "2배 부풀려짐" | ⚠️ 설계서는 2배 단정, 제안서는 완화 표현 — **제안서가 오히려 더 안전한 표기** |
| "전환 이벤트는 단 1개(conversion)만 측정 중" | 설계서 동일 | ✅ 일치 |
| "고객이 장바구니에 담고 이탈하는지… 어떤 상품이 팔리는지" | 이하 문장 절단 | ⚠️ 문장 미완성 — 클라이언트 제출 불가 상태 |

```
현재 조건에서 사용 가능한가: 조건부YES (수치 표현은 설계서보다 안전하나 문장 미완성)
리스크:
  - 제안서 마지막 문장 절단 → 제출 시 비전문성 노출
  - 패키지 선택 "스탠다드"라고 명시했으나 스탠다드의 deliverable이 이 문서 내에 없음
현재 조건에서 보완할 것:
  → 절단된 문장 복구 위치: "어떤 상품이 팔리는지" 이하 → 마무리 문장 완성
  → "스탠다드 패키지" 섹션 추가: 작업 항목·납기·단가 명시 (설계서 보완 이후 연동)
  → 보완 위치: 제안서 본문 하단 절단 지점부터
```
