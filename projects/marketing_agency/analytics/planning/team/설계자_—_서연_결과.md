## 전제 조건 요약 (설계 근거)

| 발견 결함 | 비즈니스 영향 |
|-----------|--------------|
| GA4 ID 2개 중복 설치 | 세션·전환 수치 2배 부풀려짐 → 모든 의사결정 기준값 오염 |
| 전환 이벤트 1개만 존재 (`conversion`) | 퍼널 어디서 이탈하는지 모름 → 광고비 최적화 불가 |
| 이커머스 dataLayer 미구조화 | GA4·Meta Pixel 둘 다 구매 데이터 못 읽음 → ROAS 계산 근거 없음 |
| view_item / add_to_cart / purchase 미측정 | 화해 수상 이후 유입 증가분이 매출로 연결되는지 검증 불가 |

## 패키지 구성

### PACKAGE A — 기반 정비 패키지 `"데이터를 처음부터 제대로"`

> **대상:** 지금 당장 광고를 집행 중이거나, 화해 수상 이후 유입이 늘었는데 성과를 측정 못 하고 있는 상황

---

#### 포함 작업 목록

| # | 작업명 | 세부 내용 | 이것이 없으면 발생하는 손실 |
|---|--------|-----------|----------------------------|
| A-1 | GA4 중복 ID 제거 및 단일화 | `G-598429` 구버전 ID 제거, `G-07V7RKPJ1M` 단일 운영 체계 확립, GTM 내 트리거 정리 | 세션이 2배로 집계되어 전환율·이탈률 수치가 구조적으로 틀린 상태로 방치됨 |
| A-2 | GA4 이커머스 이벤트 세팅 (필수 5종) | `view_item` / `add_to_cart` / `begin_checkout` / `add_payment_info` / `purchase` — GTM dataLayer 기반으로 구현 | 퍼널 이탈 지점 모름 → 광고 예산 어디 써야 할지 근거 없음 |
| A-3 | dataLayer 이커머스 스키마 설계 | 상품 ID·상품명·카테고리·가격·수량·주문번호 포함 구조 정의, 개발팀 전달용 스펙 문서 포함 | 이벤트를 심어도 '어떤 상품이 팔렸는지' 속성값이 비어 분석 불가 |
| A-4 | Meta Pixel 구매 이벤트 연동 | `Purchase` 이벤트에 `value` + `currency` 파라미터 추가, GA4 purchase와 동일 dataLayer 활용 | Meta 광고 ROAS가 추정값이 아닌 실측값 0원으로 보임 → 캠페인 최적화 신호 없음 |
| A-5 | GA4 전환 이벤트 재설정 | 기존 `conversion` 이벤트 검토 후 `purchase` 를 핵심 전환으로 등록, 보조 전환(begin_checkout) 추가 설정 | 광고 최적화 알고리즘이 엉뚱한 행동을 전환으로 학습함 |
| A-6 | GTM 컨테이너 정리 | 미사용 태그·트리거·변수 감사(Audit) 후 제거, 네이밍 규칙 통일 | 태그 충돌·중복 실행으로 이벤트 발화 오류 재발 위험 상존 |
| A-7 | 검증 리포트 납품 | GA4 DebugView + GTM Preview 캡처, 이벤트별 정상 발화 확인서, 수정 전/후 비교 스크린샷 포함 | 세팅 완료 여부를 담당자가 직접 확인할 수 없어 차후 분쟁 발생 |

**예상 작업 시간:** 18~24시간 (개발팀 dataLayer 반영 일정에 따라 ±3시간)

| 지표 | 세팅 전 | 세팅 후 기대값 |
|------|---------|----------------|

> **보수적 추정:** Meta·GA4 최적화 신호 정상화 시, 동일 예산 대비 광고 ROAS 15~30% 개선 (뷰티 D2C 업종 평균 기준, 퍼널 이벤트 세팅 후 3개월 내)

### PACKAGE B — 성과 분석 패키지 `"어디서 돈이 나오고 어디서 새는지"`
