**일시:** 2026-04-19

## 지금 이 순간에도 손실이 발생하고 있습니다

> **화해 뷰티어워드 3관왕.**
> 인스타그램 팔로워 20,300명.
> 히디프는 지금 브랜드 인지도가 가장 빠르게 오르는 시기입니다.
>
> 문제는, **이 유입이 매출로 연결되는지 지금 아무도 모른다**는 것입니다.

히디프의 웹사이트를 진단한 결과, **세 가지 구조적 결함**이 확인되었습니다.

| 발견된 문제 | 지금 일어나고 있는 일 |
|------------|----------------------|
| GA4 측정 ID 2개 동시 가동 중 | 세션·전환이 실제보다 많게 집계 → 모든 ROAS 수치가 부풀려진 채 의사결정에 쓰이고 있음 |
| 구매·장바구니·결제 이벤트 전무 | 광고를 클릭한 고객이 '구매했는지'를 아무 채널도 보고받지 못하고 있음 |
| 구매 데이터 구조 미완성 | Meta 광고가 '무엇이 팔렸는지' 학습하지 못한 채 예산을 소진 중 |

**이 상태로 광고비를 쓰는 것은, 영수증 없이 지갑을 여는 것과 같습니다.**

## 이 손실의 구체적 비용

> 아래 추정은 진단 데이터와 D2C 뷰티 업종 평균 지표를 기반으로 산출합니다.
> 히디프의 실제 광고비·매출 데이터가 확인되면 더 정밀한 수치를 제공할 수 있습니다.

### 문제 1. GA4 이중 집계 — "잘못된 나침반으로 항해 중"

GA4 ID가 2개 가동되면 세션과 전환이 **최대 2배로 부풀려**집니다.
예를 들어 실제 전환율이 1.0%인데 대시보드에는 2.0%로 표시됩니다.

- 이 수치를 믿고 광고 예산을 배분하면, **실제로 낮은 성과를 높다고 착각**하게 됩니다.
- 반대로 실제 잘 되는 채널이 상대적으로 낮아 보여 **예산을 줄이는 역방향 최적화**가 발생합니다.

> **지금 히디프의 광고 의사결정은 신뢰할 수 없는 수치 위에 서 있습니다.**

---

### 문제 2. 구매 이벤트 미측정 — "광고비를 쓰는데 결과를 모름"

뷰티 D2C에서 Meta 광고가 제대로 최적화되려면 **구매 이벤트 데이터를 충분히 누적**해야 합니다.
현재 히디프의 Meta Pixel은 구매 신호를 수신하지 못하고 있습니다.

- 구매 신호 없이 Meta 알고리즘은 **'클릭하는 사람'을 찾을 뿐, '사는 사람'을 찾지 못합니다.**
- 구매 데이터가 쌓이기 시작하면 동일 예산 대비 **광고 효율 30~50% 개선**이 가능합니다.
  *(스탠다드 패키지 기대 성과 기준)*

---

### 문제 3. 화해 수상 효과를 돈으로 환산하지 못하고 있음

화해 뷰티어워드 3관왕은 **강력한 마케팅 자산**입니다.
수상 이후 유입 증가, SNS 언급 증가, 검색량 증가가 발생했을 것입니다.

그러나 현재 히디프는 **이 유입이 실제 구매로 얼마나 이어졌는지 데이터가 없습니다.**

- 수상 효과를 수치로 증명할 수 없으면, 다음 마케팅 투자 판단 근거가 없습니다.
- 이 기회를 측정하지 못하는 것 자체가 **브랜드 자산을 낭비하는 것**입니다.

## 권장 패키지: 스탠다드

> **판단 근거:** GA4 중복 ID 정리 + 이커머스 이벤트 전체 재구성 + 멀티채널 어트리뷰션 분석이 동시에 필요한 상황입니다. 스타터는 이 공수를 감당하지 못하고, 프리미엄은 데이터 신뢰 기반이 갖춰진 이후 단계입니다.

#### PHASE 1 — 기반 정비 (데이터를 처음부터 제대로)

| # | 작업명 | 무엇을 하는가 | 이것이 없으면 |
|---|--------|-------------|--------------|
| A-1 | **GA4 중복 ID 제거 및 단일화** | `G-598429` 제거, `G-07V7RKPJ1M` 단일 운영 체계 확립, GTM 내 트리거 정리 | 세션·전환율이 구조적으로 2배 부풀려진 채 방치됨 |
| A-2 | **GA4 이커머스 이벤트 세팅 (필수 5종)** | `view_item` / `add_to_cart` / `begin_checkout` / `add_payment_info` / `purchase` — GTM dataLayer 기반 구현 | 퍼널 이탈 지점 모름 → 광고 예산 배분 근거 없음 |
| A-3 | **dataLayer 이커머스 스키마 설계** | 상품 ID·상품명·카테고리·가격·수량·주문번호 포함 구조 정의, 개발팀 전달용 스펙 문서 포함 | 이벤트가 심어져도 '어떤 상품이 팔렸는지' 속성값 비어 분석 불가 |
| A-4 | **Meta Pixel 구매 이벤트 연동** | `Purchase` 이벤트에 `value` + `currency` 파라미터 추가, GA4와 동일 dataLayer 활용 | Meta 광고 ROAS가 실측값 0원으로 표시 → 캠페인 최적화 신호 없음 |
| A-5 | **GA4 전환 이벤트 재설정** | 기존 `conversion` 이벤트 검토 후 `purchase`를 핵심 전환으로 등록, 보조 전환(`begin_checkout`) 추가 설정 | 광고 최적화 알고리즘이 '구매'가 아닌 엉뚱한 행동을 학습함 |
| A-6 | **GTM 컨테이너 정리** | 미사용 태그·트리거·변수 감사(Audit) 후 제거, 네이밍 규칙 통일 | 태그 충돌·이벤트 중복 발화 오류 재발 위험 |
