# 데이터 분석 서비스 — 현장 영업 가이드

> 이 문서는 영업사원용이다. 클라이언트에게 보여주는 자료가 아님.
> 미팅 전 5분, 상담 중 판단 기준, 계약 후 Day 1 체크리스트까지 전부 담겼다.

---

## 1. 미팅 전 준비 (사전 조사 — 10분이면 된다)

### 타겟 사이트에서 확인할 것 (방문 전 필수)

**Step 1 — GA4 설치 여부 확인**

1. 타겟 사이트 접속
2. Chrome 확장 프로그램 **"GA Debugger"** 또는 **"Tag Assistant"** 설치 (무료)
3. 사이트 열고 확장 실행 → GA4 태그(G-XXXXXXX) 뜨면 설치 있음
4. 안 뜨거나 UA-XXXXXXX만 뜨면 → **GA4 없음 or 구버전만 있음** (이게 스타터 진입 포인트)

**Step 2 — 메타 픽셀 설치 여부 확인**

1. Chrome 확장 **"Facebook Pixel Helper"** 실행 (무료)
2. 사이트 열었을 때 픽셀 감지되면 있는 것
3. 감지 안 되면 픽셀 없음
4. 감지되더라도 "이벤트 미감지" 표시면 → **세팅은 됐지만 이벤트 빠진 상태** (이게 스탠다드 진입 포인트)

**Step 3 — 광고 집행 여부 확인**

1. Meta 광고 라이브러리(facebook.com/ads/library) 검색 → 집행 중이면 보임
2. 구글 투명성 센터에서 브랜드 검색
3. 둘 다 없으면 아직 광고 초기 → 스타터 포지셔닝

**사전 조사 결과 메모 양식**

| 항목 | 확인 결과 | 의미 |
|------|----------|------|
| GA4 설치 | 있음 / 없음 / UA만 | |
| 메타 픽셀 | 있음 / 없음 / 이벤트 누락 | |
| Meta CAPI | 알 수 없음 (미팅에서 확인) | |
| GTM | Tag Assistant로 확인 | |
| 현재 광고 집행 채널 | 메타 / 구글 / 네이버 / 없음 | |

## 2. 첫 상담 흐름

### 원칙: 파는 게 아니라 문제를 찾아준다는 프레임

타겟은 "마케팅 대행사가 영업하러 왔다"고 생각하는 순간 방어막을 친다.
"데이터 점검해드릴게요"로 시작해서 문제를 먼저 발견해주면 신뢰가 생긴다.

---

### 오프닝 (2분)

```
"오늘 광고 데이터 현황 잠깐 여쭤봐도 될까요?
저희가 이커머스 대표님들 만나면서 보면, 광고는 열심히 집행하시는데
데이터가 제대로 잡히지 않아서 어디서 돈이 새는지 모르는 경우가 정말 많거든요.
오늘 10분만 여쭤보고, 문제가 있으면 말씀드리고, 없으면 없다고 할게요."
```

이유: 방어막 제거 + 진단 프레임 설정. "팔러 왔다"가 아니라 "봐드리러 왔다"로 시작.

---

### Situation 질문 (3분) — 현재 상황 파악

아래 순서대로 묻는다. 전부 물을 필요 없음. 2~3개만 골라서 자연스럽게.

- "지금 어떤 채널에 광고 집행하고 계세요? 메타? 구글?"
- "월 광고비가 대략 어느 정도 되세요?"
- "광고 성과 지표는 어디서 보고 계세요? 메타 광고 관리자에서만 보시나요?"

---

### Problem 질문 (3분) — 문제 인식 유도

```
"혹시 광고관리자에서 보이는 전환수랑, 실제 주문 들어오는 숫자랑 차이 나신 적 있으세요?"
```

여기서 "응, 맞아요" 나오면 Pain이 있는 것.

```
"GA4는 세팅되어 있으신가요? 세팅돼 있으면 전환 경로가 보이는데, 
없으면 어떤 광고가 실제 매출로 연결됐는지 알 방법이 없거든요."
```

```
"iOS 업데이트 이후로 메타 전환 데이터가 줄어들었다는 거 들어보셨어요?
지금 픽셀만으로는 전환의 30~40%가 안 잡히는 경우가 많아요."
```

---

### Implication 질문 (2분) — 문제의 실제 비용을 계산하게 만든다

이게 핵심이다. 타겟 스스로 손실을 계산하게 해야 계약 동기가 생긴다.

```
"월 광고비 [X]만원 집행하시는데, 데이터가 30% 누락된다고 하면
어떤 광고가 효과 있는지 모른 채 [X × 0.3 = ]만원을 쓰고 계신 셈이거든요.
이게 쌓이면 분기에 [X × 0.9 = ]만원 정도가 방향 없이 집행된 거예요."
```

예시: 월 광고비 500만원 → "한 달에 150만원 정도는 어디서 효과 났는지 모른 채 쓰는 거죠."

---

### Need-Payoff 질문 (2분) — 해결 후 모습을 그리게 한다

```
"만약 광고별로 실제 매출 연결이 되는 채널을 정확히 알 수 있으면,
어떤 채널을 늘리고 어떤 채널을 줄이실 것 같으세요?"
```

```
"매월 '이 캠페인이 매출 기여가 없다'는 리포트를 받으시면,
그 예산을 다른 데 쓰실 수 있잖아요."
```

여기서 타겟이 스스로 이득을 말하게 하면 클로징이 쉬워진다.

---

### 진단 발표 (2분) — 사전 조사 결과 공개

```
"오시기 전에 사이트 잠깐 봤는데요,
[GA4 없음 / 픽셀 이벤트 누락 / CAPI 미세팅] 상태더라고요.
이 상태면 지금 [구체적 문제] 이 발생하고 있을 거예요."
```

이 순간이 신뢰 전환점이다. 만나기 전에 이미 분석해왔다는 것 자체가 전문성 증명.

## 3. 패키지 권유 기준

### 월 광고비 기준 1차 분기

| 월 광고비 | 제안 패키지 | 핵심 제안 포인트 |
|----------|-----------|----------------|
| 300만원 미만 | 스타터 (65만원) | "GA4 + 메타 픽셀 기본 세팅, 월 리포트로 첫 데이터 기반 마련" |
| 300만~1,000만원 | 스타터 → 스탠다드 검토 | 광고채널 2개 이상이면 스탠다드 권유 |
| 1,000만~3,000만원 | 스탠다드 (150만원) | "CAPI 없으면 데이터 30% 누락 — 150만원이 회수되는 수준" |
| 3,000만원 이상 | 스탠다드 → 프리미엄 | GTM DataLayer, BigQuery, Attribution 필요성 제기 |
| 5,000만원 이상 | 프리미엄 (300만원) | "광고비의 0.6% 투자해서 50% 효율 개선" |

### 2차 분기 — 채널 복잡도로 판단

- 메타 + 구글만: 스타터로 시작 가능
- 메타 + 구글 + 네이버 + 카카오: 스탠다드 필수
- 위 4개 + 틱톡 or 앱 운영: 프리미엄

### 스탠다드로 업셀 타이밍

스타터 권유 후 이 질문 하나만 더 한다.

```
"혹시 네이버 스마트스토어나 카카오 광고도 집행하고 계세요?
그러면 사실 스타터는 메타/구글만 잡히고 나머지 채널은 못 보거든요.
채널 통합으로 보시려면 스탠다드가 더 맞아요."
```

## 4. 자주 나오는 Objection과 답변

### Objection 1: "GA4 이미 세팅되어 있어요"

틀린 대답: "아, 그럼 됐겠네요." (기회 포기)

올바른 대답:
```
"세팅은 되어 있는데, 이벤트가 제대로 잡히고 있는지 확인해보신 적 있으세요?
GA4는 설치만 해놓으면 페이지뷰 정도만 잡히고, 구매 전환이나 버튼 클릭 같은
커스텀 이벤트는 따로 설계해야 해요.
지금 GA4 데이터에서 '구매 완료' 이벤트가 몇 건 찍히는지 보여주실 수 있어요?"
```

타겟이 GA4 열어서 확인 못 하면 → "설치는 있는데 제대로 안 쓰이는 상태"로 현장 진단 가능.

---

### Objection 2: "너무 비싸요"

틀린 대답: "조금 조정해드릴 수 있어요" (가격 깎기 절대 금지)

올바른 대답:
```
"지금 월 광고비가 [X]만원이신데, 데이터가 안 잡혀서 비효율 광고를 
계속 돌리면 매달 [X의 20~30%]만원 정도는 낭비로 나가고 있을 수 있어요.
스탠다드 150만원 투자하면 그 낭비분 잡아내는 데 쓰는 비용이거든요.
3개월이면 본전 충분히 뽑히는 구조예요."
```

계산 예시를 바로 제시한다:
- 월 광고비 1,000만원 × 20% 낭비 = 200만원/월 낭비
- 스탠다드 150만원 < 절약분 200만원 → ROI 양수

---

### Objection 3: "직접 할 수 있어요"

틀린 대답: "저희가 더 잘해요" (자랑 어필은 역효과)

올바른 대답:
```
"물론 가능해요. GA4 설치 자체는 유튜브 보면 되고요.
근데 실제 현장에서 보면, 혼자 세팅하면 이벤트 이중 집계나 누락 문제가 
90%는 생겨요. 설치는 됐는데 데이터 신뢰도가 낮아서 
결국 '이 데이터 맞아?' 하면서 못 쓰는 경우가 대부분이거든요.
저희는 세팅 후 품질 감사까지 하고, 매달 리포트로 데이터 이상 여부를 체크해줘요."
```

"직접 할 수 있다"는 사람에게 싸우지 말고, "세팅 후 유지/검증"의 가치를 제시한다.

---

### Objection 4: "일단 생각해볼게요"

틀린 대답: "네, 연락주세요" (대화 종료)

올바른 대답:
```
"어떤 부분이 확신이 안 서세요?
가격인지, 효과에 대한 확신인지, 아니면 지금 당장 우선순위가 아닌 건지
구체적으로 여쭤봐도 될까요?"
```

이 질문 하나로 실제 거절 이유를 알 수 있다. 막연한 "생각해볼게요"는 거절이 아니라 정보 부족이다.

---

### Objection 5: "Meta 픽셀 있으면 충분하지 않나요?"

```
"픽셀만으로는 iOS 14 이후부터 전환의 30~40%가 잡히지 않아요.
쿠키 차단이 기본으로 되어 있는 환경에서는 픽셀(클라이언트사이드)이 
데이터를 못 보내거든요.
Meta CAPI는 서버에서 직접 전환 데이터를 메타로 보내는 방식이라,
픽셀이 놓친 전환을 서버에서 보완해주는 이중 추적이에요.
지금 광고관리자에서 전환 수가 실제 주문보다 적게 잡히면 이게 원인일 가능성이 높아요."
```

## 5. 계약 후 온보딩 Day 1 체크리스트

계약 당일 또는 다음 영업일 내 완료해야 하는 것들이다. 이걸 빠르게 챙길수록 클라이언트 신뢰가 올라간다.

### Day 1 체크리스트

**클라이언트에게 받아야 할 것 (당일)**
- [ ] GA4 계정 관리자 권한 (구글 계정 이메일 공유)
- [ ] Meta 비즈니스 관리자 접근 권한 (스탠다드 이상)
- [ ] 웹사이트 CMS 접근 권한 또는 GTM 컨테이너 접근 권한
- [ ] 광고 집행 채널 목록 (메타/구글/네이버/카카오 여부)
- [ ] 월 광고비 예산 확인 (패키지 맞는지 재확인)

**내부 세팅 (당일 시작)**
- [ ] 작업 지시서 작성 + 벤더 배정 (스타터: 크몽 풀 / 스탠다드: OSOMA or 비즈스프링)
- [ ] 클라이언트 슬랙/카카오 채널 개설
- [ ] 진행 일정표 공유 (세팅 완료 예상일 명시)
- [ ] 첫 리포트 발송 예정일 확정

**첫 커뮤니케이션 (당일)**
- [ ] 카카오 또는 이메일로 "진행 시작" 메시지 발송
  ```
  "[대표님 이름]님, 저희 팀에서 바로 작업 시작했습니다.
  [날짜]까지 [패키지 포함 항목] 세팅 완료 예정이고,
  완료 후 점검 리포트 바로 공유드릴게요.
  진행 중 문의사항은 이 채널로 언제든 주세요."
  ```

## 부록: 데이터 진단 멘트 카드 (현장 즉시 사용)

상황별 1줄 멘트. 외워두면 자연스럽게 나온다.

| 상황 | 멘트 |
|------|------|
| GA4 없음 확인 | "지금 광고 집행하시는데 성과 측정 기반이 없는 상태예요. 눈 감고 광고하는 거랑 같아요." |
| 픽셀만 있고 CAPI 없음 | "iOS 이후로 메타 전환이 최대 40% 누락되고 있을 수 있어요. 실제 효과를 과소 측정하는 거죠." |
| 데이터 있는데 못 읽는 상황 | "데이터가 있어도 읽을 수 없으면 없는 거랑 같아요. 저희 리포트가 '어디서 돈 버는지' 직접 짚어드려요." |
| 비싸다 반론 시 | "광고비의 1% 미만 투자로 나머지 99%가 더 효율적으로 쓰이는 구조예요." |
| 직접 하겠다 반론 시 | "세팅은 혼자 되는데, 유지/검증이 문제예요. 데이터 이상이 생겨도 모르고 넘어가는 게 더 큰 손해예요." |
