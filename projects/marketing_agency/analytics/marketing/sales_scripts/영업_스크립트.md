# 데이터 분석 / 트래킹 서비스 — 콜드 영업 스크립트 교과서

> 대상: 월 광고비 300만~5,000만 집행하는 이커머스 / D2C 브랜드 대표 또는 마케터
> 채널: 이메일 / 인스타 DM / 링크드인
> 목표: 진단 미팅 잡기 (가격 논의 아님)

---

## 사전 체크리스트 (발송 전 5분 — 이것 없이 보내면 스팸)

타겟 트래킹 현황을 직접 확인한다. 추정 금지.

```
✅ 1. GA4 설치 여부
   → Chrome 확장 "GA Debugger" 또는 "Tag Assistant" 로 실사이트 접속해서 확인
   → GA4 ID가 2개 이상 설치돼 있으면 이중집계 가능성 있음 (1차 DM 각도: 데이터 오염)

✅ 2. 메타 픽셀 설치 여부
   → Facebook Pixel Helper 확장으로 실사이트 확인
   → 픽셀 있는데 purchase 이벤트 없으면 "전환 신호 없음" 각도 사용
   → 픽셀 있는데 CAPI 없으면 "iOS14 손실" 각도 사용

✅ 3. GTM 설치 여부
   → 소스코드에서 "googletagmanager.com" 검색
   → 없으면 "모든 태그 하드코딩" 상황 → 유지보수 비용 각도

✅ 4. 인스타/링크드인 있는지 + 최근 광고 집행 여부
   → 페이스북 라이브러리(facebook.com/ads/library)에서 브랜드명 검색
   → 광고 집행 중이면 "지금 이 광고 낭비되고 있음" 각도 강화

✅ 5. 타겟 담당자 특정
   → 이커머스 대표 → 매출/비용 프레임
   → 마케터 → 성과 증명/ROAS 개선 프레임
   → 가능하면 링크드인에서 직책 확인

확인 후 [브랜드명], [도메인], [확인된 문제] 변수를 채운 뒤 발송.
```

# 1차 콜드 DM — Hook (손실 실체화 + 패턴 인터럽트)

**목표**: 답장 유도 (판매 아님. "이거 우리 얘기다" 반응 유도)
**심리 기제**: 패턴 인터럽트 + 손실 프레임 + "우리 사이트 직접 본 사람" 인식
**성공 지표**: 48시간 이내 답장 20% 이상
**금지**: 회사명 첫 줄 노출, 근거 없는 수치, 가격 언급, "안녕하세요 [회사명]입니다" 형식

---

## 이메일 버전 A — GA4 이중 집계 각도 (데이터 오염 프레임)

**전략**: 직접 사이트에서 발견한 기술적 문제를 구체적으로 짚는다. "우리가 네 사이트 실제로 봤다"는 것을 느끼게.

**사용 조건**: GA4 ID가 2개 이상 설치된 것이 확인됐을 때

```
제목: [브랜드명] 사이트에서 GA4 ID 2개 발견했습니다

안녕하세요.

[브랜드명] 사이트([도메인]) 트래킹 상태를 간단히 확인해봤는데,
GA4 측정 ID가 2개 설치돼 있더라고요.

이 경우 페이지뷰, 세션, 전환 데이터가 두 배로 잡힙니다.
지금 보고 있는 ROAS 수치가 실제보다 높게 나올 수 있어요.

잘못된 데이터로 광고 예산을 배분하고 있을 수 있다는 뜻이고요.

혹시 이 부분 인지하고 계셨나요?
확인하고 싶으시면 30분 짧게 통화 가능합니다.

[이름]
[연락처]
```

→ 왜 이 멘트인가: "GA4 ID 2개"는 Tag Assistant로 실측 가능한 사실이고, ROAS 왜곡이라는 직접 손실로 연결된다. 마케터에게 "내가 모르고 있던 문제"를 알려주는 구조라 방어기제가 낮아진다.

---

## 이메일 버전 B — Meta CAPI 없음 각도 (iOS 전환 손실 프레임)

**전략**: "픽셀은 있는데 CAPI 없음" 상황에서 실측 가능한 손실 수치 제시.

**사용 조건**: Facebook Pixel Helper로 픽셀 확인됐지만 purchase 이벤트 없거나 CAPI 미구축 확인 시

```
제목: [브랜드명] 메타 광고, 전환의 30~50%가 안 잡히고 있을 수 있습니다

안녕하세요.

[브랜드명] 사이트에서 메타 픽셀이 설치된 건 확인했는데,
서버사이드 전환 추적(CAPI)이 없더라고요.

iOS14 업데이트 이후로 픽셀 단독으로는
실제 구매 전환의 30~50%가 메타에 신호로 넘어가지 않습니다.

메타 광고가 "이 사람들이 구매자다"를 학습하지 못하는 상태에서
지금 광고 최적화가 제대로 될 수 없어요.

한 번 확인해볼 만한 내용이라 연락드렸습니다.
관심 있으시면 아래로 연락 주세요.

[이름]
[연락처]
```

→ 왜 이 멘트인가: "30~50% 손실"은 iOS14 이후 업계에서 검증된 수치다. 픽셀 설치는 했으나 CAPI 없는 브랜드가 대다수라 "우리 얘기"로 받아들이는 확률이 높다.

---

## 이메일 버전 C — 전환 이벤트 없음 각도 (광고 최적화 불능 프레임)

**전략**: purchase 이벤트 미설치 = 광고 알고리즘이 맹목적으로 돌아가는 상황 설명.

**사용 조건**: 픽셀 있지만 purchase/add_to_cart 이벤트 없는 것 확인 시

```
제목: [브랜드명] 메타 광고가 구매자를 모르는 상태로 돌아가고 있습니다

안녕하세요.

[브랜드명] 사이트에서 메타 픽셀은 설치됐는데,
purchase(구매) 이벤트가 잡히지 않더라고요.

메타 알고리즘은 "이 광고를 보고 실제로 산 사람"을 학습해서
비슷한 사람에게 광고를 더 보여줍니다.

지금은 그 학습 신호가 전혀 없는 상태예요.
광고비가 나가고 있는데 누가 전환했는지 알 방법이 없는 거고요.

15분만 시간 내주시면 현재 상태 정확히 진단해드릴 수 있습니다.

[이름]
[연락처]
```

→ 왜 이 멘트인가: "구매자를 모른다"는 표현이 비개발자 대표/마케터에게 직관적으로 와닿는다. 기술 용어 없이 문제의 심각성을 전달.

---

## 인스타 DM 버전 A — 짧고 직접적 (모바일 최적화)

**전략**: 인스타 DM은 긴 글을 안 읽는다. 한 줄 후킹 + 질문 하나로 압축.

**사용 조건**: 인스타 광고 집행 확인된 브랜드

```
안녕하세요. [브랜드명] 인스타 광고 보고 연락드렸어요.

혹시 메타 광고에서 "전환"이 실제 판매랑 맞아떨어진다고 생각하세요?

iOS 업데이트 이후로 픽셀만으로는 실제 구매의 절반이 안 잡히는 경우가 많아서요.
지금 [브랜드명] 광고가 어떤 상태인지 짧게 확인해드릴 수 있어요.

관심 있으시면 말씀해주세요.
```

→ 왜 이 멘트인가: 인스타 DM은 3줄 이상 읽히지 않는다. 질문 하나로 자기 상황을 떠올리게 하고, CTA는 열린 형태로 부담을 낮춘다.

---

## 인스타 DM 버전 B — 숫자 후킹 (마케터 타겟)

**사용 조건**: 마케터 직접 접촉 시 / 광고 성과에 민감한 포지션

```
안녕하세요. 데이터 트래킹 쪽 일하는데요.

[브랜드명] 사이트 잠깐 봤는데 픽셀에 purchase 이벤트가 안 잡히더라고요.

메타 광고가 구매자 학습을 못 하는 상태라 ROAS가 실제보다 낮게 나오거나
좋은 고객층에 광고가 안 가고 있을 수 있어요.

세팅 한번 같이 봐도 될까요?
```

→ 왜 이 멘트인가: 마케터는 "ROAS"라는 단어에 즉각 반응한다. 문제를 기술 용어로 짚되, 해결의 키를 쥐고 있는 사람으로 포지셔닝.

## 링크드인 버전 A — 전문적 직접 제안 (B2B 톤)

**사용 조건**: 마케팅 디렉터, 그로스 팀장, CMO 등 직급 높은 담당자

```
안녕하세요, [이름]님.

[브랜드명]의 트래킹 인프라를 간단히 살펴봤는데, 현재 CAPI 없이
픽셀만 운영 중이시더라고요.

iOS14 이후 환경에서는 픽셀 단독으로는 전환 신호가 메타에
충분히 전달되지 않아 알고리즘 최적화에 직접적인 영향이 있습니다.

월 [광고비 규모] 이상 집행하시는 브랜드에서 이 부분이 미세팅 상태면
실질적인 ROAS 손실로 연결되는 경우가 많습니다.

30분 짧게 현황 공유해드릴 수 있는데, 관심 있으시면 연락 주세요.

[이름] / [연락처]
```

→ 왜 이 멘트인가: 링크드인은 수신자가 직책자다. "기술 용어 + 비즈니스 임팩트"를 함께 쓰는 것이 신뢰도를 높인다. 단, 추정 수치 없이 조건부 표현("경우가 많습니다")을 사용.

## 링크드인 버전 B — 트래킹 감사 무료 제공 후킹

**사용 조건**: 마케팅 팀장, 대표 직접 접촉

```
안녕하세요, [이름]님.

GA4, 픽셀, GTM 전반을 점검하는 트래킹 감사 서비스를 운영하고 있는데요.

[브랜드명] 사이트 기준으로 어느 부분에서 데이터 손실이 발생하고 있는지
무료로 한 번 점검해드릴 수 있어요.

감사 결과만 보고 판단하셔도 됩니다.
부담 없이 연락 주세요.

[이름] / [연락처]
```

→ 왜 이 멘트인가: "무료 감사"는 진입 장벽을 0으로 낮춘다. 결과물을 먼저 경험하게 하는 방식이 신뢰 구축에 효과적이며, 이후 유료 전환 퍼널로 연결된다.

# 2차 DM — 진단 리포트 티저 (가치 증명)

**목표**: "이 사람들 실력 있구나" 확인 + 미팅 수락
**전제**: 1차에서 답장 받은 후, 또는 무응답 5~7일 후 재발송
**심리 기제**: 사회적 증거 + 전문성 신호 + 구체적 이익 제시

---

## 이메일 / 링크드인 버전 A — 간이 진단 결과 공유

**사용 조건**: 1차에서 관심 표현 후 다음 단계로 유도

```
제목: [브랜드명] 트래킹 간이 진단 결과 공유드립니다

안녕하세요.

말씀 이어서, [브랜드명] 사이트 트래킹 현황을 간략하게 정리해봤습니다.

[확인된 문제 1~2가지 구체적으로 기재 — 아래 예시 참고]

예시:
- GA4: 측정 ID 2개 설치 → 데이터 이중 집계 상태
- 메타 픽셀: 설치됨 / CAPI 없음 → iOS 환경 전환 손실 발생 중
- purchase 이벤트: 미설치 → 광고 최적화 신호 없음

이 3가지만 잡아도 광고 성과 판단의 정확도가 달라집니다.

30분 통화로 전체 현황 공유드리고, 어떻게 개선할 수 있는지 설명드릴게요.
이번 주 중 가능하신 시간이 있을까요?

[이름]
[연락처]
```

→ 왜 이 멘트인가: 실측 기반 문제 리스트는 "우리를 실제로 본 전문가"라는 신뢰를 만든다. 일반적인 "우리 서비스는~" 피칭보다 열람율이 높다.

---

## 이메일 버전 B — 무응답 재발송 (각도 전환)

**사용 조건**: 1차 발송 5~7일 후 무응답

```
제목: Re: [브랜드명] 트래킹 문의 — 짧게 하나만 여쭤볼게요

안녕하세요.

지난번 메일 바쁘셔서 못 보셨을 것 같아서요.

질문 하나만 드리면 — 지금 메타 광고 ROAS가 실제 매출이랑 맞아 떨어진다고
느끼세요? 광고 플랫폼에서 보이는 전환 수랑 실제 주문 수가 비슷한지요.

이 두 숫자가 많이 다르다면 트래킹 문제일 가능성이 높아요.
확인해드릴 수 있습니다.

[이름]
[연락처]
```

→ 왜 이 멘트인가: 재발송은 다른 각도로 접근해야 한다. "ROAS vs 실제 매출 불일치"는 누구나 한 번씩 느끼는 감각이라 공감 후킹이 된다.

---

## 인스타 DM 버전 — 무응답 재발송

**사용 조건**: 1차 인스타 DM 읽씹 후 4~5일 뒤

```
혹시 요즘 메타 광고 성과가 예전만 못하다는 느낌 있으세요?

광고비는 비슷한데 전환이 줄었다면 픽셀 문제인 경우가 꽤 있어요.
짧게 확인해드릴게요.
```

→ 왜 이 멘트인가: 인스타 재발송은 더 짧아야 한다. "느낌"을 건드리는 것이 열린 질문으로 작동하며 답장 부담이 낮다.

# 3차 DM — 스탠다드 패키지 제안 + CTA

**목표**: 미팅 확정 또는 제안서 수락
**전제**: 2차에서 관심 확인됐거나 미팅 이후 팔로업
**심리 기제**: 구체적 ROI 제시 + 손실 역산 + 명확한 CTA

---

## 이메일 버전 A — 스탠다드 패키지 직접 제안

**사용 조건**: 월 광고비 1,000~3,000만 확인된 브랜드

```
제목: [브랜드명] 트래킹 개선 제안 — 월 150만원으로 가능한 것들

안녕하세요.

이전에 말씀드린 것처럼 지금 [브랜드명] 트래킹 상태에서는
광고 최적화 신호가 메타에 제대로 전달되지 않고 있습니다.

저희가 스탠다드 패키지(월 120~180만원)로 해드릴 수 있는 것:
- CAPI 구축 → iOS 환경 전환 복구
- purchase 이벤트 세팅 → 광고 알고리즘 학습 정상화
- GA4 이중 집계 정리 → 실제 ROAS 파악 가능
- Looker Studio 대시보드 → 채널별 성과 한눈에 확인

이번 주 중 30분 통화로 구체적인 범위와 가격 확정하시겠어요?
[달력 링크 또는 연락처]

[이름]
[연락처]
```

→ 왜 이 멘트인가: "월 150만원"이라는 숫자 단독이 아니라 "ROAS 10% 개선 시 추가 매출"로 ROI를 역산해준다. 가격이 비용이 아닌 투자로 인식되게.

# 시퀀스 타이밍

```
Day 0 → 1차 DM 발송

Day 21 → 최종 팔로업 or 드롭

응답 있으면:
대화 → 진단 리포트 공유 → 30분 통화 → 패키지 제안 → 계약
```

# 절대 금지 사항

- 첫 DM에서 가격 언급 금지
