# 개발사 0→1 진입 계획 (dev_products entry_plan)

> **비유**: 개발 외주는 식당 새 메뉴 출시처럼 — 이미 손님이 몰려있는 플랫폼(위시켓/크몽)에 "우리 요리"를 올리고, 동시에 개인 손님(콜드메일)도 찾아가는 양손 전략이 식재 비용은 거의 안 들고 마진은 70-90%다.

**결론**: Claude Code 기반 솔로 개발 외주 사업. 위시켓/크몽 입찰 + 콜드메일 병행. 첫 12개월 목표: 월 3,500만원 (보수추정).

**이유**: (1) 위시켓 평균 프로젝트 3,270만원이지만 1-2건만 달아도 월급, (2) 자동화(automation) 카테고리는 1주 내 납기+단순코드=마진90%, (3) 한국 시장 솔로 dev-shop 선례 다수(입찰만으로 연 1억+).

**위험**: 첫 2개월 입찰 낮은 성공률(5-10%), 클라이언트 커뮤니케이션 병목, 프로젝트 overrun시 수익성 급락.

## 1. PRD (한 줄 정의)

**"Claude Code 기반 솔로 개발 외주 — 위시켓/크몽 입찰 + 콜드메일 병행, 평균 3주 인도, 첫 12개월 목표 매출 4,200만원 (보수추정: 월 3,500만원) / 12월 모니터링 후 재설정"**

- **사업자 형태**: 개인사업자 (Lian 1인)
- **핵심 상품**: 자동화 + 웹사이트 (2개 카테고리 우선)
- **수익모델**: 프로젝트 건당 선금 + 납기료
- **타겟 배급**: 위시켓/크몽(80%) + 콜드메일(20%)

## 2. ICP (누구에게 파는가)

### Primary: 입찰 플랫폼 의뢰사

| 플랫폼 | 의뢰사 특성 | 평균 예산 | 마진 예상 |
|---|---|---|---|
| **위시켓** | 중소기업 IT담당 / 스타트업 | 500만~5,000만원 | 70-80% |
| **크몽** | 소상공인 / 프리랜서 (저예산) | 50만~500만원 | 80-90% |

### Secondary: 콜드메일 ICP

**조건 3개 (모두 만족):**
1. **회사규모**: 직원 50~500명 (IT팀 있지만 개발 아웃소싱 상시 필요)
2. **산업**: 자영업 SaaS, 교육테크, 마케팅에이전시, 중소 유통사
3. **Pain**: "개발자 채용 못 함" 또는 "개발팀 스케일 한계"
