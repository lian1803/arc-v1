    🔌 기초 개념 1: API란 무엇인가?

    소프트웨어들이 대화하는 방법

    💡 한 문장 정의

    API (Application Programming Interface)는
    "이 프로그램이 저 프로그램에게 요청하고 응답받는 방법"입니다.

    🍽️ 비유: 레스토랑
    • 메뉴 (Menu) = API 문서
    • 주문 (Order) = API 요청
    • 요리사 (Chef) = AI 모델
    • 음식 (Food) = API 응답

    메뉴에 없는 것은 주문할 수 없음!
    ⚡ 실전에서
    • DeepL이 자체 번역 AI API를 호출
    • Perplexity가 GPT-4/Claude API를 호출
    • 우리는 API를 직접 쓰지 않아도 됨!
    • AI 도구들이 대신 처리

    🪙 기초 개념 3: 토큰(Token)이란?

    AI가 이해하는 텍스트의 최소 단위

    💡 한 문장 정의

    Token (토큰)은
    AI가 텍스트를 처리하는 최소 단위입니다. (영어: 약 0.75단어, 한국어: 약 0.5단어)

    📊 토큰 계산 예시
    영어:
    "Hello world" = 2 tokens
    "I love AI" = 3 tokens

    한국어:
    "안녕하세요" = 2-3 tokens
    "인공지능이 좋아요" = 5-6 tokens

    💡 한국어는 영어보다 토큰을 더 많이 사용
    ⚠️ 왜 중요한가?
    1. 비용:
    토큰 수 × 단가 = API 비용

    2. 한도 (Limit):
    • GPT-4: 8,192 tokens
    • Claude 3: 200,000 tokens

    3. 긴 영상 = 문제:
    1시간 영상 자막은
    토큰 한도 초과 가능!

    🚨 실전 팁: 토큰 절약하기

    • 긴 문서는 나눠서 처리 (청킹)
    • 불필요한 문장은 제거 후 AI에 전달
    • 요약 먼저 → 상세 분석 순서로 처리
    • Opal/NotebookLM이 자동으로 최적화해줌!

    ② 전문성 & ③ 선택 안목

    도메인 지식 + AI = 초능력 / 효율적인 도구 선택

    📊 전문성: 도메인 + AI
    당신의 전문 지식이 AI의 날개가 됩니다

    • 내 업무의 반복 패턴을 먼저 파악
    • "이 작업을 AI가 대신할 수 있을까?" 질문
    • 전문 용어와 맥락을 프롬프트에 녹이기
    ⚖️ 선택 안목: 비용 효율
    "모든 AI를 다 쓸 필요 없다"

    ❌ 간단한 작업에 비싼 도구 → 낭비
    ❌ 새 도구만 쫓아다님 → 아무것도 못함
    ✅ 작업 복잡도에 맞는 도구 선택!

    ⚠️ 효율성 체크

    • AI 없이 30분 → AI로 1시간? → AI 안 쓰는 게 낫다
    • 무료로 되는데 유료 결제? → 비용 낭비 / 배우는 시간 vs 절약되는 시간 계산하기

    ④ 돌파력 & ⑤ 주인의식

    막혔을 때 뚫는 법 + 자기주도적 학습

    🔨 돌파력: 막혔을 때
    1. 프롬프트 재구성 - 질문 방식 바꾸기
    2. 단계 분해 - 작은 단계로 쪼개기
    3. 도구 교체 - Claude, Gemini 시도
    4. 커뮤니티 활용 - 검색, 동료에게 질문
    🚀 주인의식: 자기주도
    수동적: 가르쳐주길 기다림, 포기
    자기주도: 스스로 찾아 학습, 커스터마이징, 실패를 학습 자료로!

    ✨ AI는 도구일 뿐, 주인은 나입니다

    📋 홈스 vs 프롬프터: 6단계 대응

    추리의 기술이 곧 프롬프트의 기술입니다

    홈스의 추리 설명 프롬프트 대응 실천 방법
    ① 대상 포착 추리/수사 대상 포착 AI와 협업할 거리 포착 문제 해결 거리를 구상
    ② 관찰/수집 관찰과 자료 수집 배경지식, 데이터 수집 모델의 능력/제한 파악
    ③ 적절한 질문 본질 꿰뚫는 질문 충돌 지점 파악 도메인 지식으로 추리
    ④ 가설 만들기 다양한 가능성 제시 다양한 프롬프트 시도 "이 톤은?", "포맷 바꾸면?"
    ⑤ 가설 평가 가설 검증/배제 출력 검토 및 반복 수정 Iterative Refinement
    ⑥ 결론 도출 개연성 높은 결론 최적 프롬프트 확정 재사용 템플릿으로 저장

    🧱 노드(Node)와 엣지(Edge)란?

    레고 블록처럼 조립하는 AI 워크플로우

    🧩
    노드 (Node) = 작업 블록
    비유: 레고 블록 하나하나

    역할: 특정 작업 하나만 수행
    • "유튜브 링크 받기" 블록
    • "자막 추출하기" 블록
    • "한국어로 번역하기" 블록

    특징: 각 블록은 독립적으로 작동
    ➡️
    엣지 (Edge) = 연결선
    비유: 레고 블록을 연결하는 돌기

    역할: 데이터를 다음 블록으로 전달
    • A 블록의 결과 → B 블록의 입력
    • 화살표 방향으로 데이터가 흐름

    특징: 선이 끊어지면 데이터가 안 넘어감

    💡 핵심 개념

    노드 = 일꾼 (각자 맡은 일만 함)
    엣지 = 컨베이어 벨트 (결과물을 다음 일꾼에게 전달)

    🚑 자주 묻는 오류 & 해결법

    증상 원인 해결책
    자막을 찾을 수 없음 자막이 없는 영상 'Audio to Text' (Whisper) 노드 추가
    내용이 잘림 / 멈춤 토큰(길이) 한도 초과 'Split Text' 노드로 내용을 쪼개서 처리
    없는 말을 지어냄 할루시네이션 (AI의 헛소리) Temperature 값을 0에 가깝게 낮춤 (아래 설명 참조)
    결과 형식이 깨짐 LLM이 지시 무시 프롬프트에 예시(One-shot)를 추가
    실행 버튼이 안 눌려요 노드 연결 안 됨 모든 노드가 선으로 연결되었는지 확인
    영어로 결과가 나옴 언어 미지정 프롬프트 끝에 "한국어로 답변해줘" 추가
    에러 메시지가 떠요 API 한도 초과 잠시 기다린 후 다시 시도, 또는 무료 크레딧 확인
    결과가 너무 짧아요 지시가 모호함 "최소 500자 이상" 등 길이 조건 추가

    💡 Temperature 값이 뭐예요?

    Temperature = AI의 "창의성 조절 다이얼"
    • 0에 가까울수록: 정확하고 일관된 답변 (사실 기반 작업에 적합)
    • 1에 가까울수록: 다양하고 창의적인 답변 (아이디어 브레인스토밍에 적합)
    → 요약/번역 같은 작업은 0.2~0.3을 추천해요!

    # 강사 평가 (Instructor Perspective) 수강생 평가 (Student Perspective)
    1 [노드/워크플로우] Opal에서 노드와 엣지의 개념을 이해하고, Input→Process→Output 흐름을 설계할 수 있는가? 노드를 레고 블록처럼 연결해서 나만의 자동화 앱을 만들 수 있게 되었다.
    2 [멀티 AI 전략] ChatGPT, Claude, Gemini 등 각 AI의 강점을 파악하고 상황에 맞게 조합해서 사용하는가? 한 가지 AI에 의존하지 않고, 업무에 따라 최적의 AI 조합을 선택할 수 있다.
    3 [RAG 원리] NotebookLM의 작동 원리(내가 올린 소스만 참조)를 이해하고 할루시네이션 최소화 전략을 적용하는가? 내가 올린 문서만 기반으로 답변하니 더 신뢰할 수 있고, 출처도 확인 가능하다.
    4 [Audio Overview] NotebookLM의 팟캐스트 기능을 활용하고, Customize로 역할/청중/톤을 지정할 수 있는가? 어려운 문서도 "초등학생용 팟캐스트"로 변환해서 출퇴근길에 가볍게 들을 수 있다.
    5 [심화 프롬프트] "독창적 지식 분리", "단계별 학습", "다중 소스 비교" 등 고급 프롬프트 전략을 적용하는가? 단순 요약을 넘어서, 여러 문서의 공통점/차이점과 독창적 인사이트를 뽑아낸다.
    6 [AI 에이전트] Manus, Claude Code 등 에이전트 AI의 개념을 이해하고 "대화→실행" 패러다임 전환을 인식하는가? AI가 답변만 하는 게 아니라 직접 작업을 실행해주는 시대가 왔음을 체감했다.
    7 [MCP] Model Context Protocol의 개념을 이해하고, AI가 외부 도구와 연결되는 원리를 설명할 수 있는가? AI가 Notion, Slack, 브라우저 등 내 도구들과 연결되어 진짜 비서처럼 일하는 원리를 알게 됐다.
    8 [아이디어 팩토리] AI를 창의적 파트너로 활용하여 브레인스토밍, 역발상, 페르소나 기법을 적용하는가? 막혔을 때 AI에게 아이디어를 쏟아내게 하고, 내가 선별하는 역할 분담이 가능해졌다.
    9 [워크플로우 설계] 여러 AI 도구를 체계적으로 연결하여(수집→분석→생성) 하나의 업무 흐름을 완성했는가? Comet으로 수집 → NotebookLM으로 분석 → 결과물 생성까지 한 번에 끝나는 워크플로우가 생겼다.
    10 [보안과 윤리] AI 사용 시 개인정보 보호, 저작권, 환각 검증 등 윤리적 가이드라인을 실천하는가? 강력한 도구일수록 책임감 있게 사용해야 함을 알고, 결과물은 반드시 검증한다.
