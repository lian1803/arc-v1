Part 1 - 고객 가설

고객 가설 (Customer Hypothesis)

정의: "누가 이 문제를 겪고 있는가?"를 구체적으로 정의하는 것

왜 Early Adopter를 찾아야 하는가?

  • 모든 사람을 타겟하면 아무도 만족시킬 수 없다
  • Early Adopter = 문제를 가장 심하게 겪고 있어서 불완전한 솔루션도 기꺼이 사용할 사람
  • 이들의 피드백이 제품의 방향을 결정한다

구체적인 예시

🍜
배달앱
"혼자 사는 20-30대 직장인 중 매일 저녁 뭐 먹을지 고민하는 사람"
🏋
운동앱
"작심삼일로 운동을 포기하는 30대 직장인"
💡

Key Point

"20대 여성" 같은 인구통계학적 정보만으로는 부족합니다. 행동과 상황을 함께 정의하세요.

Part 1 - 문제 가설

문제 가설 (Problem Hypothesis)

정의: 고객이 겪고 있는 Top 3 문제를 정의하는 것

"문제를 이해하지 못하면 올바른 솔루션을 만들 수 없다"

- Ash Maurya

예시: 배달앱의 Top 3 문제

1

메뉴 선택 장애

매일 저녁 뭘 먹을지 결정하는 데 너무 많은 시간과 에너지를 소모한다

2

배달비 부담

음식값 + 배달비를 합치면 부담이 크다. 특히 1인분 주문 시 배달비 비율이 높다

3

배달 시간 불확실

예상 시간보다 늦게 오거나, 실시간 위치를 정확히 알 수 없다

기존 대안(Existing Alternatives) 파악이 중요!

고객이 현재 이 문제를 어떻게 해결하고 있는지 파악해야 합니다. 기존 대안이 없다면, 문제가 충분히 심각하지 않을 수 있습니다.

비즈니스 가설 (Business Hypothesis)

💵
Revenue Streams
어떻게 돈을 벌 것인가?

구독, 수수료, 광고, 프리미엄, 라이선스 등

📊
Cost Structure
핵심 비용은 무엇인가?

개발비, 서버비, 인건비, 마케팅비, 물류비 등

📏
Key Metrics
어떤 지표로 성공을 측정?

MAU, 전환율, 리텐션, LTV, CAC 등

🛡
Unfair Advantage
쉽게 복제할 수 없는 경쟁 우위

독점 데이터, 네트워크 효과, 브랜드, 특허, 커뮤니티 등

Part 2 - AI 분석

이미 존재하는 서비스의 유저저니를 AI에게 분석시켜 보세요

당신은 10년차 UX 디자이너입니다. [서비스명]의 사용자 여정을 분석해주세요. 다음 형식으로 정리해주세요: | 단계 | 행동 | 터치포인트 | 감정 | 페인포인트 | 개선 기회 | 단계는 다음을 포함해주세요: 인지 → 고려 → 온보딩 → 첫 사용 → 정기 사용 → 추천/이탈 각 단계마다 구체적인 사용자 행동과 감정을 포함해주세요.

Pro Tip: 스크린샷 첨부

  • 앱의 주요 화면 스크린샷 3-5개를 함께 첨부하면 훨씬 정확한 분석이 가능합니다
  • ChatGPT, Claude 모두 이미지 분석을 지원합니다
  • "이 스크린샷들을 바탕으로 사용자 여정을 분석해주세요"라고 요청하세요

Part 2 - AI 피드백

AI로 유저저니 피드백 받기

작성한 유저저니를 AI에게 보여주고 피드백을 받으세요

제가 작성한 사용자 여정 맵을 분석하고 개선해주세요. [여기에 작성한 유저저니 테이블을 붙여넣기] 다음 관점에서 피드백해주세요: 1. 빠진 단계나 터치포인트가 있는지 2. 각 단계의 페인포인트가 충분히 구체적인지 3. 추가로 고려해야 할 페인포인트 4. 개선 효과가 가장 클 Top 3 지점 5. 각 Top 3 지점에 대한 구체적 개선 방안

AI 활용 워크플로우

  • Step 1: 내가 직접 유저저니를 작성 (AI 없이)
  • Step 2: AI에게 피드백 요청 (빠진 부분 발견)
  • Step 3: 피드백 반영하여 수정
  • Step 4: 수정된 버전을 다시 AI에게 검증 요청

Part 3 - 핵심 능력

AI 시대에 필요한 4가지 핵심 능력

AI를 잘 다루기 위해 우리에게 필요한 것들

❓
1. 질문 능력
좋은 질문이 좋은 결과를 만든다. AI에게 명확하고 구체적으로 요청하는 힘

역할 부여, 맥락 제공, 구체적 요구사항, 제약 조건, 결과물 형태를 포함한 프롬프트 작성 능력

🏷
2. 키워드를 다루는 힘
스타일, 분위기, 트렌드를 키워드로 조합하여 원하는 결과를 이끌어내는 능력

"미니멀 + 뉴트로 + 파스텔" 같은 키워드 조합으로 디자인 방향을 정확히 전달

3. 실행력과 실험정신
빠르게 여러 버전을 만들고 비교하는 습관. 완벽보다 속도!

같은 화면을 3가지 스타일로 30분 안에 만들어보고, 가장 좋은 것을 선택하는 방식

❤️
4. 인간에 대한 깊은 이해
결국 사용자는 사람. 페르소나 기반 사고와 공감 능력이 핵심

"50대 부모님도 쉽게 쓸 수 있도록..." 같은 사용자 중심 프롬프팅

Part 3 - 키워드

키워드 조합의 힘

디자인 의도를 정확히 전달하는 공식

스타일 + 분위기 + 컬러 + 레이아웃 = 완벽한 결과
스타일 (Style)
  • Minimal (미니멀)
  • Material (매터리얼)
  • Glassmorphism (글래스)
  • Neumorphism (뉴모피즘)
  • Flat (플랫)
분위기 (Mood)
  • Professional (전문적인)
  • Playful (장난기 있는)
  • Luxurious (고급스러운)
  • Trustworthy (신뢰감)
  • Energetic (활기찬)
컬러 (Color)
  • Pastel (파스텔)
  • Vivid (선명한)
  • Dark Mode (다크모드)
  • Monochrome (단색)
  • Gradient (그라데이션)
레이아웃 (Layout)
  • Grid System (그리드)
  • Card UI (카드형)
  • Split Screen (분할)
  • Full Screen (전체화면)
  • Sidebar (사이드바)
"Create a Minimal and Professional dashboard UI with Dark Mode and Card Layout."

Part 3 - 원칙

반드시 지켜야 할 디자인 원칙

AI가 만들어줘도, 사람을 위해 검수해야 합니다

👀
1. 접근성 (Accessibility)
누구나 사용할 수 있어야 합니다.
  • 글자 크기가 충분히 큰가?
  • 배경과 글자의 명도 대비가 확실한가?
  • 색맹 사용자도 구분 가능한가?
📏
2. 일관성 (Consistency)
학습 비용을 줄여줍니다.
  • 버튼 모양이 통일되어 있는가?
  • 같은 기능은 같은 위치에 있는가?
  • 용어가 통일되어 있는가?
🗣️
3. 피드백 (Feedback)
내 행동에 시스템이 반응해야 합니다.
  • 버튼을 눌렀을 때 눌린 효과가 있는가?
  • 로딩 중에 로딩 표시가 나오는가?
  • 에러 발생 시 이유를 알려주는가?

Part 3 - 비교

좋은 UI vs 나쁜 UI

상황나쁜 UI ❌좋은 UI ✅
회원가입 입력 항목이 10개 넘고, 다 입력하고 나서야 에러를 알려줌 꼭 필요한 정보만 입력받고, 입력 즉시 유효성 검사 (실시간 피드백)
버튼 "확인", "취소" 버튼의 색상과 크기가 똑같아서 헷갈림 주요 행동(CTA) 버튼은 눈에 띄는 색상, 취소는 회색이나 테두리만
에러 메시지 "오류가 발생했습니다. (Error 404)" "페이지를 찾을 수 없어요. 주소를 다시 확인해주세요." (친절한 설명 + 해결책)
로딩 아무 반응 없이 화면이 멈춤 스켈레톤(Skeleton) UI나 스피너로 진행 중임을 표시
🤖

AI에게 검수 요청하기

"이 디자인 시안에서 UX 원칙에 위배되는 요소를 찾아줘"라고 물어보면 AI가 훌륭한 리뷰어가 되어줍니다.

Part 3 - 모바일 퍼스트

모바일 퍼스트 (Mobile First)

작은 화면부터 디자인해야 하는 이유

Why Mobile First?

  • 핵심에 집중: 화면이 좁아서 정말 중요한 기능만 남기게 됨
  • 트래픽: 이미 모바일 트래픽이 PC를 넘어섬 (대부분의 서비스)
  • 확장성: 작은 것 → 큰 것으로 확장은 쉽지만, 반대는 어려움
"Design the mobile version first. Screen width: 375px. Use a hamburger menu for navigation. Stack elements vertically. Ensure touch targets are at least 44x44px."
Touch Target - 손가락 터치 영역 확보
Readable Text - 최소 16px 이상
Simple Nav - 복잡한 메뉴 숨기기
