Part 1 - 문제 가설
문제 가설 (Problem Hypothesis)
정의:
고객이 겪고 있는 Top 3 문제를 정의하는 것
"문제를 이해하지 못하면 올바른 솔루션을 만들 수 없다"
- Ash Maurya
예시: 배달앱의 Top 3 문제
1
메뉴 선택 장애
매일 저녁 뭘 먹을지 결정하는 데 너무 많은 시간과 에너지를 소모한다
2
배달비 부담
음식값 + 배달비를 합치면 부담이 크다. 특히 1인분 주문 시 배달비 비율이 높다
3
배달 시간 불확실
예상 시간보다 늦게 오거나, 실시간 위치를 정확히 알 수 없다
기존 대안(Existing Alternatives) 파악이 중요!
고객이 현재 이 문제를 어떻게 해결하고 있는지 파악해야 합니다. 기존 대안이 없다면, 문제가 충분히 심각하지 않을 수 있습니다.

| 단계 | 행동 | 터치포인트 | 감정 | 페인포인트 | 개선 기회 |

Pro Tip: 스크린샷 첨부

"이 스크린샷들을 바탕으로 사용자 여정을 분석해주세요"라고 요청하세요

1. 질문 능력

4. 인간에 대한 깊은 이해
결국 사용자는 사람. 페르소나 기반 사고와 공감 능력이 핵심

Part 3 - 비교
좋은 UI vs 나쁜 UI
상황
나쁜 UI ❌
좋은 UI ✅
회원가입
입력 항목이 10개 넘고, 다 입력하고 나서야 에러를 알려줌
꼭 필요한 정보만 입력받고, 입력 즉시 유효성 검사 (실시간 피드백)
버튼
"확인", "취소" 버튼의 색상과 크기가 똑같아서 헷갈림
주요 행동(CTA) 버튼은 눈에 띄는 색상, 취소는 회색이나 테두리만
에러 메시지
"오류가 발생했습니다. (Error 404)"
"페이지를 찾을 수 없어요. 주소를 다시 확인해주세요." (친절한 설명 + 해결책)
로딩
아무 반응 없이 화면이 멈춤
스켈레톤(Skeleton) UI나 스피너로 진행 중임을 표시
🤖
AI에게 검수 요청하기
"이 디자인 시안에서
UX 원칙에 위배되는 요소
를 찾아줘"라고 물어보면 AI가 훌륭한 리뷰어가 되어줍니다.

A/B 테스트와 AI 활용
정답이 없을 땐, 두 가지 모두 테스트해보세요.
A/B 테스트란?
두 가지 버전(A안, B안)을 만들어 동시에 테스트하고, 더 성과가 좋은 쪽을 선택하는 방법입니다. AI에게 처음부터 두 가지 시안을 요청하면 효과적입니다.
Create TWO versions of a landing page hero section.

Version A: Large hero image with centered text overlay.
Version B: Split layout with text on left, image on right.

Both should have the same CTA button 'Start Free Trial'.
Which version would likely convert better for
[target audience]
?
실습 과제
7분
가장 중요한 화면(랜딩페이지 등)을 하나 고르세요.
AI에게 서로 다른 레이아웃의 두 가지 버전을 요청해보세요.

AI 페르소나 사용성 테스트
AI를 가상의 사용자로 빙의시켜 피드백을 받아보세요.
🤖
페르소나 설정의 힘
"같은 디자인이라도
10대 학생
과
50대 주부
의 반응은 다릅니다. 다양한 페르소나를 AI에게 부여하세요."
You are a
[persona: 55-year-old who is not tech-savvy]
.
Look at this UI design. Walk me through:

1) What do you think this app does?
2) What would you tap first?
3) Is anything confusing?
4) Would you trust this app with your personal information?

Be honest and specific.
실습 과제
10분
본인 서비스의 타겟 페르소나와, 정반대되는 페르소나를 설정하세요.
두 페르소나에게 각각 디자인에 대한 피드백을 요청하여 비교해보세요.
